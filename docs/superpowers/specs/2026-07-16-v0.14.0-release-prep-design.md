# Publishing-Ready for `vscode-beads-rust` (v0.14.0) — Design

**Date:** 2026-07-16  
**Status:** Approved (pending written-spec review)  
**Branch:** `release/v0.14.0` (off `origin/main`)  
**Tag:** `v0.14.0`  
**Target:** VS Code Marketplace + Open VSX + GitHub Releases

## 1. Summary

The extension now works against the `br` (beads_rust) CLI instead of upstream `bd`. Before tagging v0.14.0 we will: (a) rebase the 9 unpushed commits onto a fresh release branch, (b) rename the package identity from the upstream `planet57.vscode-beads` to the fork's true identity `phillipmerritt.vscode-beads-rust`, (c) rewrite user-facing documentation (README, CHANGELOG) so it matches reality, (d) update supporting project tooling docs and agent instructions to use `br` instead of `bd`, (e) ship the existing CI + release GitHub Actions workflows against the renamed package, (f) audit `.gitignore` and tracked files so we don't accidentally publish the wrong artifacts.

This is a single release-scope change: one package.json metadata rename, one README rewrite, one CHANGELOG entry freeze, one `AGENTS.beads.md` rewrite, one workflow manifest refresh, one tag. No new features. No package-lock reseed beyond what removing `mysql2` requires.

## 2. Goals

- Publish `phillipmerritt.vscode-beads-rust` v0.14.0 to VS Code Marketplace + Open VSX + a GitHub Release.
- Update the README so a new user can install, run `br init`, and use the extension without hitting documentation that still describes `bd`/Dolt.
- Update `CHANGELOG.md` so `[0.14.0]` accurately captures the `bd → br` backend switch.
- Replace all bd-tooling references in repo-local agent/docs materials with br equivalents (the fork ships a `br`-only backend).
- Pass local validation (`bun install`, `bun run lint`, `bun run test`, `bun run compile`, `bun run package` smoke) before the tag is pushed. CI is the second line of defense.
- Confirm the tracked-file surface (`git ls-files` + `.gitignore`) is exactly what we want shipped, before vsce packages it.

## 3. Non-goals (out of scope)

- Implementing the eleven `beads_ext-*` follow-up backlog items (D1–D12). Those are post-v0.14.0.
- Replacing the workflow `npx @vscode/vsce` / `npx ovsx` calls with pinned versions. Tracked separately.
- Marketplace publisher branding (logo, banner, domain verification, sponsor). Account-level, outside the repo.
- Forks of fork-documentation in `beads_ext/docs/*.md` porter scratch — that lives at the workspace level, not in the published extension.

## 5. Validation Strategy

### 5.1 Local mandatory (run before tag)

```bash
bun install --frozen-lockfile        # fail loudly if lockfile drift
bun run lint                          # zero errors, zero warnings
bun run test                          # all Jest suites green
bun run compile                       # extension.js + webview/main.js built
bun run package                       # produces vscode-beads-rust-0.14.0.vsix

# Verify shipped identity in the produced VSIX
unzip -p vscode-beads-rust-0.14.0.vsix extension/package.json | jq '{name, displayName, version, publisher}'

# Verify allowed-path set
unzip -Z1 vscode-beads-rust-0.14.0.vsix | sort | \
  diff -u - <(printf 'extension.vsixmanifest\n[Content_Types].xml\nextension/CHANGELOG.md\nextension/LICENSE\nextension/LICENSE.txt\nextension/changelog.md\nextension/dist/extension.js\nextension/dist/webview/main.css\nextension/dist/webview/main.js\nextension/docs/images/beads-vscode-screenshot.png\nextension/package.json\nextension/readme.md\nextension/resources/beads-icon.svg\nextension/resources/icon.png\n' | sort)
```

### 5.2 CI backstop

The existing `ci.yml` runs lint + compile + package on every push and PR. `release.yml` runs lint + compile + package + `vsce publish` + `ovsx publish` on tag push.

If `vsce publish` returns a non-zero exit because `VSCE_PAT` is wrong or `phillipmerritt` is owned, that's a real signal — we want it to surface before the tag is cut, ideally via the dry-run package step on the release branch.

### 5.3 Optional smoke

- Install the produced VSIX into a `code-server` instance for a manual smoke pass (per `docs/code-server-testing.md`). Non-blocking for tag.

## 6. Error Handling & Rollback

- **Marketplace rejection (publisher already taken)** — `vsce publish` exits non-zero; the release workflow must not silently succeed. We pin the step to `--fail` semantics by not adding `|| true` and not setting `continue-on-error`.
- **Open VSX rejection (namespace conflicts)** — same; pin exit codes.
- **Wrong tag version** — release.yml precondition (`tag version == package.json version`) blocks.
- **CHANGELOG missing entry** — release.yml precondition blocks.
- **Wrong publisher in package.json** — caught by CI lint + manual inspection of `unzip -p .../extension/package.json`.
- **Bad lockfile** — caught by `bun install --frozen-lockfile`.

Rollback plan if a publish goes wrong after the tag:

1. Yank the VSIX via `vsce unpublish phillipmerritt.vscode-beads-rust` (only works within 24h on the Marketplace; Open VSX has no unpublish, only `ovsx delete`).
2. Mark the GitHub Release as a pre-release / draft so users don't auto-update.
3. Cut a v0.14.1 tag with the fix.
